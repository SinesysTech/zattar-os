**Logica de fetch** (per D-09 e Pitfall 4 — paralelo):
```typescript
// Disparar ambos em paralelo ao montar
const [logs, setLogs] = useState<LogAlteracao[]>([]);
const [isLoading, setIsLoading] = useState(true);

useEffect(() => {
  let cancelled = false;
  setIsLoading(true);
  auditLogService.getLogs('audiencias', audienciaId)
    .then(data => { if (!cancelled) setLogs(data); })
    .catch(err => console.error('Timeline fetch error:', err))
    .finally(() => { if (!cancelled) setIsLoading(false); });
  return () => { cancelled = true; };
}, [audienciaId]);
```

**Montagem da timeline** (computado com useMemo):

1. **Entradas de `logs`**: Para cada `log`, extrair `changes` de `log.dados_evento.changes` (formato: `{ campo: { old, new } }`). Mapear campo para label legivel via `AUDIENCIA_FIELD_LABELS[campo] ?? campo`. Formatar valores: booleanos como "Sim"/"Nao", null como "—", strings como estao.

2. **Entrada de `dados_anteriores`** (system update PJe): Se `audiencia.dadosAnteriores` nao for null, fazer diff com estado atual:
   - Converter `dados_anteriores` (snake_case) para comparacao: usar `Object.entries(audiencia.dadosAnteriores)` e mapear cada chave via `AUDIENCIA_FIELD_LABELS`
   - Para cada campo em `dadosAnteriores` com valor diferente do atual, criar uma entrada de change
   - IMPORTANTE (Pitfall 3): `dadosAnteriores` esta em snake_case. Para comparar, mapear cada chave snake para o campo correspondente em camelCase do `audiencia` atual. Usar mapeamento simples via `AUDIENCIA_FIELD_LABELS` para lookup de chave, nao comparacao de valor.
   - Type da entrada: `'system'`, timestamp: `audiencia.updatedAt`, descricao: "Sincronizacao PJe"

3. **Entrada de captura inicial**: Sempre incluir como ultima entrada — `{ type: 'captura_inicial', timestamp: audiencia.createdAt, descricao: 'Captura inicial', changes: [] }`

4. **Ordenar** todas as entradas por timestamp descrescente (mais recente primeiro, per D-10).
